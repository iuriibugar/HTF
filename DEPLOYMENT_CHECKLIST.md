# ✅ Чек-лист перед розгортанням (Deployment Checklist)

Видаліть цей файл, якщо він вам не потрібен. Це напоминання для коректного розгортання з оновленням версії.

## Перед деплоєм

Виконайте ці кроки ДО того як розгортати:

### 1. Оновіть версію
```bash
# Відредагуйте файл:
public/version.json

# Та змініть версію (семантичне версіонування):
# 1.0.0 → 1.0.1 (патчі для мініх виправлень)
# 1.0.0 → 1.1.0 (мінор для нових фич)
# 1.0.0 → 2.0.0 (мажор для великих змін)
```

### 2. Оновіть timestamp
```json
{
  "version": "1.0.1",
  "timestamp": "2026-02-12T12:00:00Z",  // ← оновіть на поточний час
  "notes": "Опис змін"
}
```

### 3. Запустіть білди та тести
```bash
npm run type-check
npm run test:e2e  # (опціонально)
npm run build
```

### 4. Скажіть користувачам
Коли розгортання готово, оновіть документацію або новини про цю версію.

## Розгортання

### На GitHub Pages
```bash
npm run deploy
```

### На Firebase
```bash
npm run deploy:firebase
```

## Після деплою

### Перевірка
1. Перейдіть на сайт і дайте сторінці 5 хвилин
2. Перевірте DevTools Console для помилок
3. Спробуйте відкрити інший браузер (щоб симулювати нового користувача)

### Моніторинг
- Перевіряйте, чи користувачі бачать popup через 5 хвилин
- За потреби можна запустити вручну в console браузера:
```javascript

// (як користувач це робить, коли я вже знаходжусь на сайті)

// Це спрацює правильно якщо версія в version.json новіша

// Потім оновлення видалить весь кеш та обновить сторінку
```

## Семантичне версіонування (SemVer)

Форма: `MAJOR.MINOR.PATCH`

```
1.0.0
│ │ └─ PATCH: мініі виправлення, баги (1.0.1)
│ └─── MINOR: нові функції, зміни назад-сумісні (1.1.0)
└───── MAJOR: великі розламуючі зміни (2.0.0)
```

## Приклади

```
Виправлення баги              → 1.0.0 → 1.0.1
Нові фіч, нічого не ламаємо  → 1.0.0 → 1.1.0
Повна переробка              → 1.0.0 → 2.0.0
```
